                      michaelherman Mod Leo Simmons • 3 years ago

                      If you're practicing TDD, then you would -

                      1) Write an e2e test and watch it fail
                      2) Write an integration test and watch it fail
                      3) Write a unit test and watch it fail
                      4) Get the unit test to pass
                      5) Watch the integration test fail
                      6) Repeat steps 3, 4, and 5 until the integration test to pass
                      7) Watch the e2e test fail
                      8) Repeat steps 2, 3, 4, 5, 6, 7 until the e2e test passes

                      Unit tests serve to better understand why integration tests fail and integration tests serve to better understand why e2e tests fail. If you have a large app then it could take a long time for the full e2e test suite to run. Developers can get a quick sanity check with unit tests since they run a lot faster.

                      Hope that helps!
